# Минимальная версия GopiAI

## Описание
Это минимальная версия приложения GopiAI, содержащая только основной функционал:
- Главное окно
- Текстовый редактор
- Меню "Файл" с функциями открытия и сохранения

Данная версия создана как стартовая точка для пошаговой пересборки проекта, позволяющая избежать хаоса и потери работоспособности.

## Запуск

Для запуска минимальной версии приложения:

```bash
python run.py
```
или запустите файл `run_minimal.bat`

## Восстановление

Для возврата к полной версии приложения:

```bash
python restore.py
```
или запустите файл `restore.bat`

## Функции

### Работа с файлами
- **Открыть**: Позволяет открыть текстовый файл для редактирования
- **Сохранить**: Сохраняет текущий файл
- **Сохранить как...**: Сохраняет текущий файл с новым именем

## Особенности
- Все действия меню выводят информацию в консоль для отладки
- Простой и минималистичный интерфейс
- Работает независимо от основного приложения

## Требования
- Python 3.7+
- PySide6

## Структура файлов
- `minimal_app.py` - Основной файл приложения
- `run.py` - Скрипт для запуска минимальной версии
- `restore.py` - Скрипт для возврата к полной версии
- `README.md` - Этот файл с инструкциями

## Стратегия пересборки проекта

Данная минимальная версия служит основой для полной пересборки проекта. Стратегия включает:

### 1. Создание карты зависимостей
- Анализ существующего кода и составление карты зависимостей между модулями
- Документирование критических компонентов и их взаимодействий
- Выделение "ядра" приложения

### 2. Разработка плана миграции
- Разбиение проекта на чёткие этапы миграции
- Определение последовательности шагов от минимального приложения до полного функционала
- Создание чек-листов для контроля результатов каждого этапа

### 3. Система бэкапов
- Использование автоматизированных бэкапов с понятным версионированием
- Добавление метаданных к каждому бэкапу (какие функции работают)
- Быстрое переключение между версиями для тестирования

### 4. Прототипирование компонентов
- Разработка минимальных версий каждого UI-компонента отдельно
- Тестирование компонентов изолированно перед интеграцией
- Документирование особенностей работы компонентов

### 5. Тестовые индикаторы
- "Индикаторы здоровья" — логи и проверки в ключевых точках
- Простые автотесты для базовых функций
- Визуальные индикаторы состояния в интерфейсе

### 6. Управление техническим долгом
- Поддержка списка известных проблем и компромиссов
- Маркировка временных решений для будущего рефакторинга
- Фиксация решений о пропуске или переносе функций

### 7. Документирование
- Краткая документация после каждого успешного этапа
- Журнал изменений с объяснением принятых решений
- Руководства по эксплуатации для промежуточных версий

### 8. Структура проекта
- Чёткая организация каталогов с информативными README
- Система именования файлов, отражающая их функциональность
- Настройка путей импорта для избежания циклических зависимостей
