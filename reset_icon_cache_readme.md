# Инструкция по обновлению иконок в GopiAI

Если иконки в приложении отображаются некорректно (пустые места, ошибки или отсутствующие иконки), вы можете выполнить следующие действия для их обновления:

## Быстрое решение

1. Закройте приложение GopiAI полностью
2. Запустите скрипт `reset_icon_cache.py` из корня проекта:

```bash
python reset_icon_cache.py
```

3. Снова запустите приложение GopiAI

## Ручное исправление

Если быстрое решение не помогло, попробуйте следующие шаги:

1. Закройте приложение GopiAI полностью

2. Убедитесь, что библиотека Lucide установлена и директория с иконками существует:
   ```bash
   # Проверьте наличие иконок в node_modules
   ls node_modules/lucide/dist/svg
   ```

3. Если иконки в node_modules отсутствуют, переустановите библиотеку Lucide:
   ```bash
   npm install lucide --save
   ```

4. Скопируйте иконки из `node_modules` в директорию проекта:
   ```bash
   # Создаем директорию, если её нет
   mkdir -p assets/icons/lucide

   # Копируем все иконки SVG
   cp node_modules/lucide/dist/svg/*.svg assets/icons/lucide/
   ```

5. Удалите возможный кеш иконок (если он существует):
   ```bash
   rm -rf .cache/icons/*
   rm -rf assets/icons/cache/*
   ```

6. Запустите приложение GopiAI

## Дополнительные варианты

Если всё еще есть проблемы с иконками:

1. Проверьте, правильно ли указаны имена иконок в `app/ui/icon_adapter.py`. Имена в ICON_NAME_MAPPING должны соответствовать именам SVG-файлов из библиотеки Lucide.

2. Проверьте наличие ошибок в логах приложения.

3. При запуске приложения должно появиться сообщение о количестве загруженных иконок в логах.

## Поиск альтернативных имен иконок

Если вам нужно найти правильное имя иконки:

1. Смотрите в официальной документации Lucide: https://lucide.dev/icons/

2. Проверьте имена файлов в директории с иконками:
   ```bash
   ls assets/icons/lucide/ | grep "keyword"
   ```

3. Используйте диалог просмотра иконок в приложении GopiAI через меню "Help" → "Icons" → "Browse Icons".

## Отладка

Для отладки проблем с иконками можно добавить вывод лога:

```python
import logging
logging.basicConfig(level=logging.DEBUG)
```

в начало скрипта `reset_icon_cache.py` перед его запуском для получения более подробной информации.
