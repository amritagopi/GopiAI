# Проблемы с темами в GopiAI

## Обнаруженные проблемы

1. **Неопределенные переменные в QSS файлах**:
   - Переменные `#3C3C3C_color`, `#555555_hover`, `#3498DB_color`, `#6A6A6A_color`, `#444444_radius` не определены
   - Причина: в файлах QSS использованы переменные без определений

2. **Проблема несоответствия имен свойств в ThemeManager**:
   - В MainWindow._update_themes_menu используется `current_theme`, но в ThemeManager это свойство называется `current_visual_theme`
   - Вызывает ошибку AttributeError при обновлении меню тем

3. **Множественные темы в компилированной директории**:
   - В директории compiled есть разные версии файлов тем: `dark-theme.qss`, `light-theme.qss`, `fixed-dark-theme.qss` и т.д.
   - В выпадающем списке показываются лишние темы, которые на практике не работают

4. **Старые файлы тем сохранены в проекте**:
   - Файлы `DARK-theme.qss` и `LIGHT-theme.qss` заменены на `dark.qss` и `light.qss`, но старые версии остались
   - Вызывает путаницу при компиляции тем и применении стилей

## Решения

1. **Для неопределенных переменных в QSS**:
   - Добавлены определения переменных в начало файлов `dark.qss` и `light.qss`
   - Добавлены определения в функцию `_generate_stylesheet` в ThemeManager для гарантии их наличия

2. **Для проблемы несоответствия имен**:
   - Необходимо исправить `current_theme` на `current_visual_theme` в файле main_window.py

3. **Для множественных тем**:
   - Необходимо очистить директорию compiled и оставить только актуальные файлы тем
   - Рассмотреть возможность удаления устаревших файлов тем из проекта

4. **Для старых файлов тем**:
   - Удалить устаревшие файлы `DARK-theme.qss` и `LIGHT-theme.qss`
   - Обновить скрипты компиляции, чтобы они работали только с актуальными файлами
