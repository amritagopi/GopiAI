# Операции с файлами в редакторе

В редакторе кода (`CodeEditor`) реализованы следующие методы для работы с файлами:

1. `set_new_file()` - создает новый пустой файл
   - Очищает текст редактора
   - Устанавливает file_path в None
   - Сбрасывает флаг модификации документа
   - Обновляет заголовок вкладки на "Untitled"

2. `open_file(file_path)` - открывает существующий файл
   - Проверяет существование файла
   - Загружает его содержимое
   - Устанавливает file_path
   - Сбрасывает флаг модификации
   - Обновляет заголовок вкладки на имя файла

3. `save_file()` - сохраняет текущий файл
   - Если файл еще не сохранен, вызывает save_file_as()
   - Записывает содержимое редактора в файл
   - Сбрасывает флаг модификации

4. `save_file_as()` - сохраняет файл под новым именем
   - Показывает диалог выбора пути сохранения
   - Записывает содержимое в новый файл
   - Обновляет file_path
   - Сбрасывает флаг модификации
   - Обновляет заголовок вкладки

5. `has_unsaved_changes()` - проверяет наличие несохраненных изменений
   - Возвращает состояние флага модификации документа

Эти методы вызываются из MainWindow при выполнении соответствующих действий меню или панели инструментов (например, _new_file, _open_file, _save_file, и т.д.).

В проекте используется дуализм редакторов (CodeEditor и MultiEditorWidget), что иногда вызывает путаницу при реализации логики работы с файлами. Важно убедиться, что методы для работы с файлами реализованы в обоих классах.
