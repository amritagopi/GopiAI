# Исправление системы автоматической проверки подключений

## Проблема
После реорганизации кода из одного файла `main_window.py` в модульную архитектуру с миксинами, система автоматической проверки подключений сигналов перестала работать. Это произошло потому, что проверка запускалась в `MainWindowCore` до того, как миксины были применены, и поэтому система не могла проверить и подключить сигналы, добавляемые миксинами.

## Решение
1. Обновлен метод `_auto_connect_signals` в `main_window_core.py`, чтобы он работал аналогично оригинальному в `main_window.py.backup`:
   - Функция теперь показывает сообщение в статусной строке
   - Возвращает количество подключенных сигналов и информацию о подключениях
   - Обрабатывает ошибки и логирует их

2. Добавлены функции `initialize_signals` и `_run_signal_checks` в `main_window.py`:
   - `initialize_signals` запускает отложенную проверку сигналов через QTimer.singleShot
   - `_run_signal_checks` запускает автоматическое подключение сигналов и показывает результаты

3. Обновлен файл `main.py` для вызова функции `initialize_signals` после создания и показа главного окна

4. Обновлена функция `fix_connections` в `debug_ui.py` для запуска проверки подключений из диалога отладки

## Результат
Система автоматической проверки подключений теперь работает с новой модульной архитектурой. После запуска приложения она корректно анализирует все объекты интерфейса, их сигналы и возможные обработчики, подключает те, которые может подключить автоматически, и выводит информацию об остальных в лог.
