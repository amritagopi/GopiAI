# Инструмент для проверки сигналов Qt

В приложении добавлен инструмент для выявления и исправления неподключенных сигналов Qt. Эта утилита позволяет находить сигналы, которые объявлены, но не подключены ни к одному слоту, что может привести к некорректной работе приложения.

## Основные файлы

- `app/utils/signal_checker.py` - основной модуль с классами для проверки сигналов
- `app/utils/__init__.py` - обновлен для экспорта утилиты проверки сигналов
- `app/ui/main_window.py` - добавлено меню отладки с функцией проверки сигналов

## Ключевые классы и функции

1. `SignalConnectionChecker` - основной класс для проверки сигналов
   - `check_signal_connections()` - рекурсивно проверяет сигналы объектов
   - `suggest_connections()` - предлагает возможные подключения
   - `connect_signals_automatically()` - автоматически подключает сигналы

2. `SignalMonitor` - класс для мониторинга отправленных и принятых сигналов
   - Отслеживает сигналы во время выполнения приложения
   - Выявляет сигналы, которые были отправлены, но не были получены

3. Вспомогательные функции:
   - `check_main_window_signals()` - проверяет сигналы в главном окне
   - `auto_connect_signals()` - автоматически подключает сигналы

## Интеграция в UI

В главное окно добавлено меню "Отладка" в разделе "Инструменты" со следующими функциями:
- "Проверить сигналы" - запускает анализ приложения на наличие неподключенных сигналов
- При обнаружении неподключенных сигналов предлагается опция "Подключить автоматически"

## Паттерны подключения сигналов

Автоматическое подключение использует следующие шаблоны именования:
- `clicked` -> `on_clicked`, `handle_click`
- `triggered` -> `on_triggered`, `handle_action`
- `textChanged` -> `on_text_changed`, `text_updated`
- `valueChanged` -> `on_value_changed`, `value_updated`
- `message_sent` -> `handle_message`, `on_message_received`
- `insert_code_to_editor` -> `insert_code`, `on_code_insert`
- `run_code_in_terminal` -> `execute_command`, `run_command`

Для других сигналов используется шаблон `on_[имя_сигнала]`
