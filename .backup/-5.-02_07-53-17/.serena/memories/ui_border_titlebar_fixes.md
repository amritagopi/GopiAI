# Исправления для отображения окон и применения тем

В ходе проверки приложения были обнаружены и исправлены следующие проблемы:

## 1. Прозрачный диалог цветовой гармонии

**Проблема:**
Диалог настройки цветовой гармонии (ColorHarmonyDialog) отображался с полностью прозрачным фоном.

**Решение:**
- Добавлен метод `_apply_dialog_style()`, который применяет соответствующие стили к диалогу в зависимости от текущей темы
- Метод устанавливает явные цвета фона, текста и рамки для диалога
- Добавлены также стили для метки и кнопок внутри диалога
- Стили учитывают текущую тему (светлую или темную)

## 2. Синяя обводка и черная верхняя панель главного окна

**Проблема:**
Рамка главного окна и верхняя панель заголовка не меняли цвет при смене темы, оставаясь синей и черной соответственно.

**Решение:**
- В файлах тем `dark.qss` и `light.qss` добавлены стили для рамки главного окна, использующие переменную `@border_color`
- В стилях для MainWindowTitleBar заменены жестко заданные цвета на переменные из темы:
  - `@titlebar_background` вместо #1e1e1e/#f0f0f0
  - `@titlebar_text` вместо #ffffff/#212121
- Теперь рамка окна и заголовок изменяют свой цвет в соответствии с выбранной пользовательской темой

## Общие изменения
1. Изменены файлы `dark.qss` и `light.qss` для использования переменных темы вместо жестко заданных цветов
2. Обновлен класс `ColorHarmonyDialog` для корректного отображения
3. Сохранена совместимость с существующей системой тем

Эти изменения обеспечивают согласованный внешний вид всех элементов приложения при смене темы, включая границы окна и заголовок.
