# Архитектура плагинной системы GopiAI

## Общая структура
Мы создали плагинную систему для GopiAI, которая позволяет расширять функциональность базового "замороженного" приложения без изменения его исходного кода.

Система состоит из следующих компонентов:
- Менеджер плагинов (`plugins/plugin_manager.py`)
- Базовые классы плагинов (`plugins/core/base_plugin.py`)
- Пример плагина закругленных углов (`plugins/rounded_corners/plugin.py`)
- Новый основной файл запуска приложения (`app.py`)
- Документация по созданию плагинов (`PLUGINS.md`)

## Принцип работы
1. Новое приложение (`app.py`) загружает "замороженное" приложение из директории `1_minimal_version_BACKUP_PERFECT`
2. Подключает систему плагинов, которая может модифицировать поведение приложения через хуки
3. Плагины могут добавлять новые функции, изменять внешний вид и т.д., не изменяя исходный код
4. Каждая новая функциональность добавляется как отдельный плагин

## Типы плагинов
1. `BasePlugin` - базовый абстрактный класс для всех плагинов
2. `MenuPlugin` - для добавления новых пунктов меню
3. `UIPlugin` - для добавления новых элементов интерфейса
4. `ThemePlugin` - для добавления новых тем оформления

## Точки расширения (хуки)
- `after_stylesheet_applied` - вызывается после применения стилей
- `window_created` - вызывается после создания главного окна

## Пример плагина
Для демонстрации создан плагин `RoundedCornersPlugin`, который добавляет закругленные углы к интерфейсу.

## Инструкции по запуску
Приложение с плагинами запускается через `run_with_plugins.bat` или напрямую `python app.py`.
