# Стратегия инкрементальных изменений

При работе с большими файлами или сложными изменениями:

1. Сначала проанализировать файл и создать план изменений
2. Разделить изменения на малые, независимые части
3. Применять изменения последовательно, коммитя после каждого шага
4. Использовать временные файлы для промежуточных результатов
5. Предпочитать точечные изменения через search_replace вместо полного переписывания файла
