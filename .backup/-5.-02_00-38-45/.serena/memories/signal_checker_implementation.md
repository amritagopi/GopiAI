# Инструмент проверки сигналов

В проекте реализован инструмент для обнаружения неподключенных сигналов Qt. Этот инструмент помогает выявить проблемы с сигналами и слотами, которые могут привести к некорректной работе приложения.

## Основные файлы

- `app/utils/signal_checker.py` - основной модуль с инструментами проверки сигналов
- `app/utils/__init__.py` - экспортирует классы и функции для использования в других модулях 
- `app/ui/main_window.py` - интеграция инструмента в интерфейс пользователя

## Ключевые компоненты

1. **SignalConnectionChecker** - основной класс для проверки подключений сигналов
   - Рекурсивно исследует все QObject в приложении
   - Выявляет неподключенные сигналы с помощью метода isSignalConnected
   - Может автоматически подключать сигналы по паттернам именования

2. **SignalMonitor** - класс для динамического мониторинга сигналов
   - Отслеживает отправленные, но не полученные сигналы
   - Полезен для диагностики проблем во время выполнения приложения

3. **Вспомогательные функции**:
   - `check_main_window_signals()` - проверяет сигналы в главном окне
   - `auto_connect_signals()` - автоматически подключает неподключенные сигналы

## Интеграция в UI

В главном окне приложения добавлено:
- Меню "Отладка" в разделе "Инструменты"
- Пункт "Проверить сигналы" для запуска анализа
- Диалог с информацией о найденных неподключенных сигналах
- Кнопка "Подключить автоматически" для исправления проблем

## Использование

1. В меню выберите: "Инструменты" -> "Отладка" -> "Проверить сигналы"
2. Если обнаружены неподключенные сигналы, вы увидите диалог с подробной информацией
3. При желании можно автоматически исправить проблемы, нажав кнопку "Подключить автоматически"

## Паттерны подключения

Автоматическое подключение основано на следующих паттернах именования:
- `clicked` -> `on_clicked`, `handle_click`
- `triggered` -> `on_triggered`, `handle_action`
- `textChanged` -> `on_text_changed`, `text_updated`
- `insert_code_to_editor` -> `insert_code`, `on_code_insert`
- `run_code_in_terminal` -> `execute_command`, `run_command`
- и т.д.

Для нестандартных сигналов используется шаблон `on_{signal_name}`.
