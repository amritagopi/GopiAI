# Принципы интерфейса приложения GopiAI

## Основные принципы

Интерфейс приложения должен быть максимально удобным для пользователя:
- Никаких сложных терминов или настроек
- Всё должно быть описано простым языком, ясно и логично
- Навигация должна быть интуитивно понятной

## Структура интерфейса

### Левая часть — Файловый проводник:
- Закреплен у левого края, не может быть отсоединен
- Может изменять размер вправо, но всегда остается слева

### Правая часть — ИИ-чат:
- Закреплен у правого края, не может быть отсоединен
- Может изменять размер влево (становиться шире), но всегда остается справа

### Нижняя часть — Терминал:
- Закреплен внизу и слева
- Можно изменять ширину и высоту, но нельзя отсоединять или перемещать
- Имеет вкладочную структуру, подобно браузеру

### Над терминалом — Текстовый редактор:
- Закреплен сверху и слева, изменяет размер в обоих направлениях
- Также использует вкладки, как браузер
- Текстовые вкладки имеют один цвет, а другие типы вкладок — другие цвета, должны быть визуально различимы

## Поведение окон

Все окна (кроме файлового проводника, чата, терминала и текстового редактора) могут:
- Становиться плавающими, или
- Прикрепляться сверху (рядом с текстовым редактором), или
- Прикрепляться снизу (рядом с терминалом)

Логика вкладок должна оставаться согласованной во всех случаях.

## Визуальное оформление

### Цветовая схема
- Должна быть приятна для глаз при длительной работе
- Поддерживать как светлую, так и темную тему
- Иметь достаточный контраст для удобного чтения текста

### Шрифты
- Использовать современные шрифты без засечек
- Размер шрифта должен быть настраиваемым
- По умолчанию использовать шрифт Inter для интерфейса

### Иконки
- Использовать современные иконки с единым стилем
- Иконки должны быть интуитивно понятны
- Поддерживать различные размеры экрана

## Принципы взаимодействия

- Все важные действия должны быть доступны через интерфейс
- Должны поддерживаться горячие клавиши для основных операций
- Интерфейс должен предоставлять обратную связь для действий пользователя
- Должна быть возможность отменить большинство действий
