# Разрешение конфликтов в директориях utils

## Обнаруженные проблемы

1. Конфликт в импорте ThemeManager в `app/ui/utils/__init__.py`:
   - Была попытка импортировать из локального модуля, которого не существует: `from .theme_manager import ThemeManager`
   - Это приводило к ошибкам при импорте из `app.ui.utils`

2. Потенциальное дублирование функциональности:
   - `app/utils/theme_utils.py` - утилиты для работы с темами
   - `app/ui/utils/css_tools` - инструменты для работы с CSS/QSS файлами
   - Не было четкого разделения ответственности между этими модулями

## Выполненные исправления

1. Исправлен импорт ThemeManager в `app/ui/utils/__init__.py`:
   - Заменено `from .theme_manager import ThemeManager` на `from app.utils.theme_manager import ThemeManager`
   - Добавлены также импорты CSS-инструментов для удобства использования

2. Созданы README файлы для документирования назначения модулей:
   - `app/utils/README.md` - описывает основные модули в app/utils и их назначение
   - `app/ui/utils/README.md` - объясняет назначение инструментов в app/ui/utils и их взаимодействие с другими модулями

3. Установлено четкое разделение ответственности:
   - `app/utils/theme_manager.py` и `app/utils/theme_utils.py` - управление темами
   - `app/ui/utils/css_tools` - инструменты для обработки CSS/QSS

## Рекомендации по дальнейшему использованию

1. Используйте `ThemeManager` из `app.utils.theme_manager` для всех операций с темами
2. Используйте инструменты из `app.ui.utils.css_tools` только для обработки CSS/QSS файлов на этапе разработки
3. Следуйте документации в README файлах при работе с модулями
4. Избегайте создания дублирующейся функциональности
