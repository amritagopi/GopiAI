# Исправление контекстного меню для добавления пункта "Insert Emoji"

## Проблема
В стандартном контекстном меню базового класса `CodeEditor` (app/ui/widgets.py) отсутствовал пункт "Insert Emoji".

## Решение
1. Добавлен метод `contextMenuEvent` в базовый класс `CodeEditor` в файле `app/ui/widgets.py` для переопределения стандартного контекстного меню и добавления пункта "Insert Emoji".
2. Добавлен метод `_show_emoji_dialog` для вызова диалога выбора эмодзи.
3. Исправлены методы `exec_()` на `exec()` для соответствия стилю основного проекта.
4. Использована локализация для текста "Insert Emoji" через функцию `tr("menu.insert_emoji", "Insert Emoji")`.

## Технические детали
- Класс `CodeEditor` в `widgets.py` является базовым классом, который используется в `central_widget.py`.
- Стандартное меню создается с помощью `super().createStandardContextMenu()`.
- Для показа диалога используется метод `_show_emoji_dialog(position)`.
- Результат диалога проверяется через `result == QDialog.Accepted`.
- Используется иконка "smile" из менеджера Lucide иконок.
